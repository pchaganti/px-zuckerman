## Contribution Guidelines

### Before You Start

- Check existing issues and pull requests to avoid duplicate work
- For larger changes, consider opening an issue first to discuss the approach
- Keep changes focused and atomic

### Making Changes

1. **Fork and clone** the repository
2. **Create a branch** from `main` with a descriptive name
3. **Make your changes** â€” remember: deletion > addition
4. **Test your changes** â€” ensure existing functionality still works
5. **Commit** with clear, descriptive messages
6. **Push** and open a pull request

### Code Style

- Follow existing code patterns and conventions
- Keep functions small and focused
- Prefer explicit over implicit
- Remove dead code, unused imports, and commented-out code
- Write code that's easy to delete later

### Pull Request Process

- Provide a clear description of what changed and why
- Highlight any code that was removed or simplified
- Ensure all tests pass
- Update documentation if needed

## Questions?

Feel free to open an issue or start a discussion. We're here to help!

Remember: **Less code is better code. Deletion is the best contribution.**
